# 개발 워크플로

> 이 파일은 [common/git-workflow.md](./git-workflow.md)를 확장하여 git 작업 이전에 이루어지는 전체 기능 개발 프로세스를 설명합니다.

기능 구현 워크플로는 개발 파이프라인을 기술합니다: 리서치, 계획 수립, TDD, 코드 리뷰, 그리고 git 커밋.

## 기능 구현 워크플로

0. **리서치 및 재사용** _(새 구현 전 필수)_
   - **먼저 GitHub 코드 검색:** 새로운 내용을 작성하기 전에 `gh search repos`와 `gh search code`로 기존 구현, 템플릿, 패턴을 찾습니다.
   - **리서치에 Exa MCP 활용:** 계획 단계에서 `exa-web-search` MCP를 사용하여 광범위한 리서치, 데이터 수집, 선행 사례를 발굴합니다.
   - **패키지 레지스트리 확인:** 유틸리티 코드를 작성하기 전에 npm, PyPI, crates.io 등의 레지스트리를 검색합니다. 직접 구현보다 검증된 라이브러리를 선호합니다.
   - **재사용 가능한 구현 탐색:** 문제의 80% 이상을 해결하는 오픈소스 프로젝트를 찾아 포크, 포팅 또는 래핑 가능한지 확인합니다.
   - 요구사항을 충족하는 경우, 처음부터 새로 작성하기보다 검증된 접근법을 채택하거나 포팅하는 것을 선호합니다.

1. **먼저 계획 수립**
   - **planner** agent를 사용하여 구현 계획 생성
   - 코딩 전 계획 문서 생성: PRD, 아키텍처, 시스템 설계, 기술 문서, 작업 목록
   - 의존성과 위험 요소 파악
   - 단계별로 분해

2. **TDD 접근법**
   - **tdd-guide** agent 사용
   - 먼저 테스트 작성 (RED)
   - 테스트 통과를 위한 구현 (GREEN)
   - 리팩토링 (IMPROVE)
   - 80% 이상 커버리지 확인

3. **코드 리뷰**
   - 코드 작성 직후 **code-reviewer** agent 사용
   - CRITICAL 및 HIGH 이슈 처리
   - 가능한 경우 MEDIUM 이슈도 수정

4. **커밋 및 푸시**
   - 상세한 커밋 메시지 작성
   - conventional commits 형식 준수
   - 커밋 메시지 형식 및 PR 프로세스는 [git-workflow.md](./git-workflow.md) 참조
