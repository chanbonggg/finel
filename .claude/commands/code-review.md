# 코드 리뷰

커밋되지 않은 변경 사항에 대한 보안 및 품질 종합 리뷰:

1. 변경된 파일 확인: git diff --name-only HEAD

2. 변경된 각 파일에 대해 다음을 검사합니다.

**보안 문제 (CRITICAL):**
- 하드코딩된 자격 증명, API 키, 토큰
- SQL 인젝션 취약점
- XSS 취약점
- 입력 유효성 검사 누락
- 취약한 의존성
- 경로 탐색 위험

**코드 품질 (HIGH):**
- 50줄 초과 함수
- 800줄 초과 파일
- 4단계 초과 중첩 깊이
- 에러 처리 누락
- console.log 구문
- TODO/FIXME 주석
- 공개 API에 대한 JSDoc 누락

**모범 사례 (MEDIUM):**
- 가변 패턴 (불변 패턴 사용 권장)
- 코드/주석에 이모지 사용
- 새 코드에 대한 테스트 누락
- 접근성 문제 (a11y)

3. 다음 내용으로 리포트 생성:
   - 심각도: CRITICAL, HIGH, MEDIUM, LOW
   - 파일 위치 및 줄 번호
   - 문제 설명
   - 수정 제안

4. CRITICAL 또는 HIGH 문제 발견 시 커밋 차단

보안 취약점이 있는 코드는 절대 승인하지 않습니다!
