# Refactor Clean

테스트 검증을 통해 데드 코드를 안전하게 식별하고 제거합니다.

## 1단계: 데드 코드 탐지

프로젝트 유형에 따라 분석 도구 실행:

| 도구 | 탐지 항목 | 명령 |
|------|-----------|------|
| knip | 미사용 exports, 파일, 의존성 | `npx knip` |
| depcheck | 미사용 npm 의존성 | `npx depcheck` |
| ts-prune | 미사용 TypeScript exports | `npx ts-prune` |
| vulture | 미사용 Python 코드 | `vulture src/` |
| deadcode | 미사용 Go 코드 | `deadcode ./...` |
| cargo-udeps | 미사용 Rust 의존성 | `cargo +nightly udeps` |

도구를 사용할 수 없는 경우, Grep으로 import가 없는 export 찾기:
```
# export를 찾은 후 어디서도 import되지 않는지 확인
```

## 2단계: 발견 항목 분류

안전성 등급으로 정렬:

| 등급 | 예시 | 조치 |
|------|------|------|
| **안전** | 미사용 유틸리티, 테스트 헬퍼, 내부 함수 | 확신을 갖고 삭제 |
| **주의** | 컴포넌트, API 라우트, 미들웨어 | 동적 import 또는 외부 사용자 없는지 확인 |
| **위험** | 설정 파일, 진입점, 타입 정의 | 수정 전 조사 필요 |

## 3단계: 안전 삭제 루프

각 안전 항목에 대해:

1. **전체 테스트 실행** — 기준선 확립 (모두 통과)
2. **데드 코드 삭제** — Edit 도구로 정밀 제거
3. **테스트 재실행** — 아무것도 깨지지 않았는지 확인
4. **테스트 실패 시** — 즉시 `git checkout -- <file>`로 되돌리고 해당 항목 건너뜀
5. **테스트 통과 시** — 다음 항목으로 이동

## 4단계: 주의 항목 처리

주의 항목 삭제 전:
- 동적 import 검색: `import()`, `require()`, `__import__`
- 문자열 참조 검색: 설정에서 라우트 이름, 컴포넌트 이름
- 공개 패키지 API에서 export되는지 확인
- 외부 사용자 없는지 확인 (배포된 경우 의존성 확인)

## 5단계: 중복 통합

데드 코드 제거 후 다음을 찾습니다:
- 유사 함수 (80% 이상 유사) — 하나로 병합
- 중복 타입 정의 — 통합
- 가치를 추가하지 않는 래퍼 함수 — 인라인 처리
- 목적 없는 재export — 간접 참조 제거

## 6단계: 요약

결과 보고:

```
데드 코드 정리
──────────────────────────────
삭제됨:   미사용 함수 12개
           미사용 파일 3개
           미사용 의존성 5개
건너뜀:   2개 항목 (테스트 실패)
절감:     ~450줄 제거
──────────────────────────────
모든 테스트 통과 ✅
```

## 규칙

- **테스트 먼저 실행 없이 절대 삭제 금지**
- **한 번에 하나씩 삭제** — 원자적 변경으로 롤백 용이
- **불확실하면 건너뜀** — 프로덕션을 깨는 것보다 데드 코드 유지가 나음
- **정리 중 리팩토링 금지** — 관심사 분리 (먼저 정리, 나중에 리팩토링)
