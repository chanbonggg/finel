# /learn - 재사용 가능한 패턴 추출

현재 세션을 분석하고 스킬로 저장할 가치 있는 패턴을 추출합니다.

## 트리거

비사소한 문제를 해결했을 때 세션 중 아무 때나 `/learn`을 실행하세요.

## 추출 대상

다음을 찾습니다:

1. **에러 해결 패턴**
   - 어떤 에러가 발생했나요?
   - 근본 원인은 무엇인가요?
   - 무엇으로 해결했나요?
   - 유사한 에러에 재사용 가능한가요?

2. **디버깅 기법**
   - 비직관적인 디버깅 단계
   - 효과적인 도구 조합
   - 진단 패턴

3. **우회 방법**
   - 라이브러리 특이 동작
   - API 제한사항
   - 버전별 수정사항

4. **프로젝트별 패턴**
   - 발견된 코드베이스 관례
   - 내린 아키텍처 결정
   - 통합 패턴

## 출력 형식

`~/.claude/skills/learned/[pattern-name].md`에 스킬 파일 생성:

```markdown
# [설명적인 패턴 이름]

**추출일:** [날짜]
**컨텍스트:** [적용 시점에 대한 간략한 설명]

## 문제
[이 패턴이 해결하는 문제 - 구체적으로]

## 해결책
[패턴/기법/우회 방법]

## 예시
[해당되는 경우 코드 예시]

## 사용 시점
[트리거 조건 - 이 스킬을 활성화해야 하는 상황]
```

## 프로세스

1. 추출 가능한 패턴이 있는지 세션 검토
2. 가장 가치 있고 재사용 가능한 인사이트 식별
3. 스킬 파일 초안 작성
4. 저장 전 사용자 확인 요청
5. `~/.claude/skills/learned/`에 저장

## 참고

- 사소한 수정 (오타, 단순 문법 오류)은 추출하지 않습니다
- 일회성 문제 (특정 API 장애 등)는 추출하지 않습니다
- 미래 세션에서 시간을 절약할 패턴에 집중합니다
- 스킬은 집중적으로 유지합니다 - 스킬당 하나의 패턴
