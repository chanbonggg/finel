# Strategic Compact 스킬

자동 압축에 의존하는 대신, 워크플로의 전략적 시점에 수동 `/compact`를 제안한다.

## 언제 사용하나

- 컨텍스트 한도에 근접하는 장시간 세션 (200K+ 토큰)
- 다단계 작업 진행 중 (조사 → 계획 → 구현 → 테스트)
- 같은 세션 내에서 관련 없는 작업으로 전환할 때
- 주요 마일스톤 완료 후 새 작업을 시작할 때
- 응답이 느려지거나 일관성이 떨어질 때 (컨텍스트 압박)

## 전략적 압축이 필요한 이유

자동 압축은 임의 시점에 발동된다:
- 종종 작업 도중에 발동되어 중요한 컨텍스트를 잃음
- 논리적 작업 경계를 인식하지 못함
- 복잡한 다단계 작업을 중단시킬 수 있음

논리적 경계에서의 전략적 압축:
- **조사 완료 후, 실행 전** — 조사 컨텍스트를 압축하고 구현 계획만 유지
- **마일스톤 완료 후** — 다음 단계를 위해 새롭게 시작
- **주요 컨텍스트 전환 전** — 다른 작업 전 조사 컨텍스트 정리

## 작동 방식

`suggest-compact.js` 스크립트가 PreToolUse (Edit/Write)에서 실행되어:

1. **도구 호출 추적** — 세션의 도구 호출 횟수 집계
2. **임계값 감지** — 설정된 임계값에서 제안 (기본값: 50회)
3. **주기적 알림** — 임계값 초과 후 25회마다 알림

## 훅 설정

`~/.claude/settings.json`에 추가:

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit",
        "hooks": [{ "type": "command", "command": "node ~/.claude/skills/strategic-compact/suggest-compact.js" }]
      },
      {
        "matcher": "Write",
        "hooks": [{ "type": "command", "command": "node ~/.claude/skills/strategic-compact/suggest-compact.js" }]
      }
    ]
  }
}
```

## 설정

환경 변수:
- `COMPACT_THRESHOLD` — 첫 번째 제안 전 도구 호출 횟수 (기본값: 50)

## 압축 결정 가이드

언제 압축할지 결정하는 표:

| 단계 전환 | 압축 여부 | 이유 |
|-----------------|----------|-----|
| 조사 → 계획 | 예 | 조사 컨텍스트는 방대함; 계획이 핵심 산출물 |
| 계획 → 구현 | 예 | 계획은 TodoWrite나 파일에 있음; 코드를 위한 컨텍스트 확보 |
| 구현 → 테스트 | 경우에 따라 | 테스트가 최근 코드를 참조하면 유지; 포커스가 바뀌면 압축 |
| 디버깅 → 다음 기능 | 예 | 디버그 흔적은 무관한 작업에 방해됨 |
| 구현 중간 | 아니오 | 변수명, 파일 경로, 부분적 상태를 잃는 비용이 큼 |
| 실패한 접근 후 | 예 | 새 접근 시도 전 막힌 논리 정리 |

## 압축 후 남는 것

무엇이 유지되는지 이해하면 자신 있게 압축할 수 있다:

| 유지됨 | 사라짐 |
|----------|------|
| CLAUDE.md 지시사항 | 중간 추론 및 분석 |
| TodoWrite 작업 목록 | 이전에 읽은 파일 내용 |
| 메모리 파일 (`~/.claude/memory/`) | 다단계 대화 컨텍스트 |
| Git 상태 (커밋, 브랜치) | 도구 호출 기록 및 횟수 |
| 디스크상의 파일 | 구두로 명시된 미묘한 사용자 선호도 |

## 모범 사례

1. **계획 후 압축** — 계획이 TodoWrite에 확정되면 압축하여 새롭게 시작
2. **디버깅 후 압축** — 오류 해결 컨텍스트 정리 후 계속 진행
3. **구현 중간에는 압축하지 않음** — 관련 변경을 위한 컨텍스트 보존
4. **제안을 참고** — 훅이 언제인지 알려주지만, 할지 말지는 사용자가 결정
5. **압축 전 저장** — 중요한 컨텍스트를 파일이나 메모리에 저장 후 압축
6. **요약과 함께 `/compact` 사용** — 커스텀 메시지 추가: `/compact Focus on implementing auth middleware next`
