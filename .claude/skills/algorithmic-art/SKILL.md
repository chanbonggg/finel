알고리즘 철학은 코드를 통해 표현되는 컴퓨테이셔널 미학 운동입니다. .md 파일(철학), .html 파일(인터랙티브 뷰어), .js 파일(제너레이티브 알고리즘)을 출력합니다.

이 작업은 두 단계로 이루어집니다:
1. 알고리즘 철학 생성 (.md 파일)
2. p5.js 제너레이티브 아트로 표현 (.html + .js 파일)

먼저 다음 작업을 수행하세요:

## 알고리즘 철학 생성

시작하기 위해, 다음을 통해 해석될 알고리즘 철학(정적 이미지나 템플릿이 아닌)을 생성합니다:
- 컴퓨테이셔널 프로세스, 창발적 행동, 수학적 아름다움
- 시드 기반 난수, 노이즈 필드, 유기적 시스템
- 파티클, 흐름, 필드, 힘
- 파라메트릭 변형과 통제된 카오스

### 핵심 이해

- 수신하는 것: 사용자의 미묘한 입력이나 지시 — 기반으로 활용하되, 창의적 자유를 제한하지 않아야 합니다.
- 생성하는 것: 알고리즘 철학/제너레이티브 미학 운동.
- 다음 단계: 같은 버전이 철학을 수신하고 코드로 표현합니다 — 90% 알고리즘 생성, 10% 핵심 파라미터로 이루어진 p5.js 스케치를 생성합니다.

다음 접근 방식을 고려하세요:
- 제너레이티브 아트 운동을 위한 선언문 작성
- 다음 단계에서 그것을 생명으로 불어넣는 알고리즘을 작성합니다

철학은 다음을 강조해야 합니다: 알고리즘적 표현. 창발적 행동. 컴퓨테이셔널 아름다움. 시드 기반 변형.

### 알고리즘 철학 생성 방법

**운동 이름 짓기** (1-2단어): "Organic Turbulence" / "Quantum Harmonics" / "Emergent Stillness"

**철학 표현** (4-6 단락 — 간결하지만 완전하게):

알고리즘적 본질을 포착하기 위해 이 철학이 어떻게 다음을 통해 나타나는지 표현하세요:
- 컴퓨테이셔널 프로세스와 수학적 관계?
- 노이즈 함수와 난수 패턴?
- 파티클 행동과 필드 역학?
- 시간적 진화와 시스템 상태?
- 파라메트릭 변형과 창발적 복잡성?

**핵심 지침:**
- **중복 피하기**: 각 알고리즘 측면은 한 번씩만 언급하세요. 새로운 깊이를 추가하지 않는 한 노이즈 이론, 파티클 역학, 수학 원리에 대한 개념을 반복하지 마세요.
- **장인 정신 반복 강조**: 철학은 최종 알고리즘이 마치 수많은 시간에 걸쳐 개발되고, 정성스럽게 다듬어지며, 절대적으로 최고 수준의 전문가가 만든 것처럼 보여야 한다고 여러 번 강조해야 합니다. 이 프레이밍은 필수적입니다 — "정교하게 제작된 알고리즘", "깊은 컴퓨테이셔널 전문성의 산물", "고통스러운 최적화", "마스터 수준의 구현" 같은 구절을 반복하세요.
- **창의적 공간 남기기**: 알고리즘 방향에 대해 구체적이되, 다음 Claude가 극도로 높은 수준의 장인 정신으로 해석적 구현 선택을 할 수 있을 만큼 충분히 간결하게 유지하세요.

철학은 다음 버전이 아이디어를 정적 이미지가 아닌 알고리즘적으로 표현하도록 안내해야 합니다. 아름다움은 과정에 있으며, 최종 프레임에 있지 않습니다.

### 철학 예시

**"Organic Turbulence"**
철학: 자연 법칙에 의해 제약된 카오스, 무질서에서 질서가 출현함.
알고리즘 표현: 레이어드 Perlin 노이즈로 구동되는 플로우 필드. 수천 개의 파티클이 벡터 힘을 따라 이동하며, 그 흔적이 유기적 밀도 맵으로 쌓임. 여러 노이즈 옥타브가 난류 영역과 고요한 구역을 생성함. 색상은 속도와 밀도에서 나타남 — 빠른 파티클은 밝게 타오르고, 느린 것은 그림자로 희미해짐. 알고리즘은 균형에 이를 때까지 실행됨 — 컴퓨테이셔널 미학의 대가가 수많은 반복을 통해 모든 파라미터를 다듬은 정교하게 조율된 균형.

**"Quantum Harmonics"**
철학: 파동과 같은 간섭 패턴을 나타내는 이산 개체들.
알고리즘 표현: 그리드에 초기화된 파티클, 각각은 사인파를 통해 진화하는 위상 값을 가짐. 파티클이 가까워지면 위상이 간섭함 — 보강 간섭은 밝은 노드를, 상쇄 간섭은 공백을 생성함. 단순 조화 운동이 복잡한 창발적 만다라를 생성함. 공명하는 아름다움을 만들기 위해 모든 비율이 신중하게 선택된 고통스러운 주파수 보정의 결과.

**"Recursive Whispers"**
철학: 스케일 전반의 자기 유사성, 유한 공간 안의 무한한 깊이.
알고리즘 표현: 재귀적으로 세분화되는 분기 구조. 각 가지는 황금비에 의해 제약되지만 약간 무작위화됨. L-시스템 또는 재귀 세분화가 수학적이면서도 유기적으로 느껴지는 나무 같은 형태를 생성함. 미묘한 노이즈 교란이 완벽한 대칭을 깨뜨림. 선 두께는 각 재귀 수준에 따라 감소함. 모든 분기 각도는 깊은 수학적 탐구의 산물.

**"Field Dynamics"**
철학: 물질에 대한 효과를 통해 보이게 된 보이지 않는 힘.
알고리즘 표현: 수학 함수나 노이즈로 구성된 벡터 필드. 가장자리에서 탄생한 파티클이 필드 라인을 따라 흐르다가 균형이나 경계에 도달하면 소멸함. 여러 필드가 파티클을 끌어당기거나, 밀어내거나, 회전시킬 수 있음. 시각화는 흔적만을 보여줌 — 보이지 않는 힘의 유령 같은 증거. 힘 균형을 통해 정교하게 안무된 컴퓨테이셔널 춤.

**"Stochastic Crystallization"**
철학: 정렬된 구조로 결정화되는 무작위 프로세스.
알고리즘 표현: 무작위화된 원 채우기 또는 보로노이 테셀레이션. 무작위 점으로 시작해서 이완 알고리즘을 통해 진화시킴. 셀이 균형에 이를 때까지 밀어냄. 셀 크기, 이웃 수, 또는 중심까지의 거리에 따라 색상 결정. 나타나는 유기적 타일링은 무작위적이면서도 필연적으로 느껴짐. 모든 시드는 독특한 결정 아름다움을 생성함 — 마스터 수준의 제너레이티브 알고리즘의 표시.

*이것들은 간략한 예시입니다. 실제 알고리즘 철학은 4-6개의 충실한 단락이어야 합니다.*

### 핵심 원칙
- **알고리즘 철학**: 코드를 통해 표현될 컴퓨테이셔널 세계관 생성
- **결과보다 과정**: 아름다움은 알고리즘 실행에서 나타난다는 것을 항상 강조 — 각 실행은 고유함
- **파라메트릭 표현**: 아이디어는 수학적 관계, 힘, 행동을 통해 소통 — 정적 구성이 아님
- **예술적 자유**: 다음 Claude가 철학을 알고리즘적으로 해석함 — 창의적 구현 공간 제공
- **순수 제너레이티브 아트**: 살아있는 알고리즘 만들기, 난수가 있는 정적 이미지가 아님
- **전문가 장인 정신**: 최종 알고리즘이 정교하게 제작되고, 수많은 반복을 통해 다듬어지며, 컴퓨테이셔널 미학 분야 절대 최고 수준의 전문가가 만든 것처럼 느껴져야 한다는 점을 반복 강조

**알고리즘 철학은 4-6 단락 길이여야 합니다.** 의도된 비전을 하나로 묶는 시적인 컴퓨테이셔널 철학으로 채우세요. 같은 점을 반복하지 마세요. 이 알고리즘 철학을 .md 파일로 출력하세요.

## 개념적 씨앗 추론

**중요 단계**: 알고리즘을 구현하기 전에 원래 요청에서 미묘한 개념적 실마리를 파악하세요.

**핵심 원칙**:
개념은 **알고리즘 자체에 내재된 미묘하고 틈새적인 참조** — 항상 직접적이지 않으며, 항상 정교합니다. 해당 주제에 익숙한 사람은 직관적으로 느낄 수 있어야 하며, 다른 사람들은 단순히 걸작 제너레이티브 구성을 경험합니다. 알고리즘 철학은 컴퓨테이셔널 언어를 제공합니다. 추론된 개념은 영혼을 제공합니다 — 파라미터, 행동, 창발 패턴에 보이지 않게 짜여진 조용한 개념적 DNA.

이것은 **매우 중요합니다**: 참조는 그 작품의 깊이를 스스로 드러내지 않고 향상시킬 만큼 충분히 정제되어야 합니다. 알고리즘적 화음을 통해 다른 곡을 인용하는 재즈 뮤지션처럼 생각하세요 — 아는 사람만이 알아차리지만, 모든 사람이 제너레이티브 아름다움을 감상합니다.

## P5.JS 구현

철학과 개념적 프레임워크가 확립되면 코드로 표현하세요. 진행하기 전에 잠시 생각을 정리하세요. 생성된 알고리즘 철학과 아래 지침만을 사용하세요.

### 단계 0: 먼저 템플릿 읽기

**중요: HTML 작성 전:**

1. Read 도구를 사용하여 `templates/viewer.html` **읽기**
2. 정확한 구조, 스타일링, Anthropic 브랜딩 **파악**
3. 해당 파일을 영감이 아닌 **실제 출발점으로 사용** — 영감만이 아님
4. **고정 섹션을 그대로 유지** (헤더, 사이드바 구조, Anthropic 색상/폰트, 시드 컨트롤, 액션 버튼)
5. 파일의 주석에 표시된 **변수 섹션만 교체** (알고리즘, 파라미터, 파라미터 UI 컨트롤)

**피해야 할 것:**
- HTML을 처음부터 새로 만들기
- 커스텀 스타일링이나 색상 체계 발명
- 시스템 폰트나 다크 테마 사용
- 사이드바 구조 변경

**따라야 할 관행:**
- 템플릿의 정확한 HTML 구조 복사
- Anthropic 브랜딩 유지 (Poppins/Lora 폰트, 밝은 색상, 그라디언트 배경)
- 사이드바 레이아웃 유지 (시드 → 파라미터 → 색상? → 액션)
- p5.js 알고리즘과 파라미터 컨트롤만 교체

템플릿이 기반입니다. 다시 만들지 말고 위에 구축하세요.

갤러리 수준의 컴퓨테이셔널 아트를 만들기 위해 알고리즘 철학을 기반으로 사용하세요.

### 기술 요구사항

**시드 기반 난수 (Art Blocks 패턴)**:
```javascript
// 항상 재현성을 위해 시드 사용
let seed = 12345; // 또는 사용자 입력에서 해시된 값
randomSeed(seed);
noiseSeed(seed);
```

**파라미터 구조 — 철학을 따르세요**:

알고리즘 철학에서 자연스럽게 나오는 파라미터를 설정하기 위해 "이 시스템의 어떤 특성을 조정할 수 있을까?"를 생각하세요.

```javascript
let params = {
  seed: 12345,  // 재현성을 위해 항상 시드 포함
  // 색상
  // 알고리즘을 제어하는 파라미터 추가:
  // - 수량 (얼마나 많이?)
  // - 스케일 (얼마나 크게? 얼마나 빠르게?)
  // - 확률 (얼마나 가능성 있게?)
  // - 비율 (어떤 비율로?)
  // - 각도 (어떤 방향으로?)
  // - 임계값 (언제 행동이 변하는가?)
};
```

**효과적인 파라미터를 설계하려면 "패턴 유형"으로 생각하는 대신 시스템이 조정 가능해야 하는 속성에 집중하세요.**

**핵심 알고리즘 — 철학 표현하기**:

**중요**: 알고리즘 철학이 무엇을 만들지 결정해야 합니다.

철학을 코드로 표현하기 위해 "어떤 패턴을 사용해야 할까?"가 아닌 "이 철학을 코드로 어떻게 표현할까?"를 생각하세요.

철학이 **유기적 창발**에 관한 것이라면 다음을 고려하세요:
- 시간이 지남에 따라 축적되거나 성장하는 요소
- 자연 법칙에 의해 제약된 무작위 프로세스
- 피드백 루프와 상호작용

철학이 **수학적 아름다움**에 관한 것이라면 다음을 고려하세요:
- 기하학적 관계와 비율
- 삼각 함수와 조화음
- 예상치 못한 패턴을 만드는 정밀한 계산

철학이 **통제된 카오스**에 관한 것이라면 다음을 고려하세요:
- 엄격한 경계 내의 무작위 변형
- 분기와 위상 전환
- 무질서에서 나타나는 질서

**알고리즘은 옵션 메뉴가 아닌 철학에서 흐릅니다.**

구현을 안내하기 위해 개념적 본질이 창의적이고 독창적인 선택을 알려주도록 하세요. 이 특정 요청에 대한 비전을 표현하는 무언가를 만드세요.

**캔버스 설정**: 표준 p5.js 구조:
```javascript
function setup() {
  createCanvas(1200, 1200);
  // 시스템 초기화
}

function draw() {
  // 제너레이티브 알고리즘
  // 정적(noLoop) 또는 애니메이션 가능
}
```

### 장인 정신 요구사항

**중요**: 마스터 제너레이티브 아티스트가 수많은 반복을 통해 탄생시킨 것처럼 느껴지는 알고리즘을 만드세요. 모든 파라미터를 신중하게 조정하세요. 모든 패턴이 목적을 가지고 나타나도록 하세요. 이것은 무작위 노이즈가 아닙니다 — 깊은 전문성을 통해 다듬어진 통제된 카오스입니다.

- **균형**: 시각적 노이즈 없는 복잡성, 경직성 없는 질서
- **색상 조화**: 무작위 RGB 값이 아닌 사려 깊은 팔레트
- **구성**: 무작위성 속에서도 시각적 위계와 흐름 유지
- **성능**: 애니메이션인 경우 부드러운 실행, 실시간 최적화
- **재현성**: 같은 시드는 항상 동일한 출력 생성

### 출력 형식

출력:
1. **알고리즘 철학** — 제너레이티브 미학을 설명하는 마크다운 또는 텍스트
2. **단일 HTML 아티팩트** — `templates/viewer.html`로 구축된 자기 완결형 인터랙티브 제너레이티브 아트 (단계 0과 다음 섹션 참조)

HTML 아티팩트는 모든 것을 포함합니다: p5.js (CDN에서), 알고리즘, 파라미터 컨트롤, UI — claude.ai 아티팩트나 모든 브라우저에서 즉시 작동하는 하나의 파일. 처음부터 시작하지 말고 템플릿 파일로 시작하세요.

### 중요: 고정 vs 변수 구분

`templates/viewer.html` 파일이 기반입니다. 필요한 정확한 구조와 스타일링을 포함합니다.

**고정 (항상 그대로 포함):**
- 레이아웃 구조 (헤더, 사이드바, 메인 캔버스 영역)
- Anthropic 브랜딩 (UI 색상, 폰트, 그라디언트)
- 사이드바의 시드 섹션:
  - 시드 표시
  - 이전/다음 버튼
  - 랜덤 버튼
  - 시드 입력 + Go 버튼으로 이동
- 사이드바의 액션 섹션:
  - 재생성 버튼
  - 리셋 버튼

**변수 (각 아트워크에 맞게 커스터마이즈):**
- 전체 p5.js 알고리즘 (setup/draw/클래스)
- 파라미터 객체 (아트에 필요한 것 정의)
- 사이드바의 파라미터 섹션:
  - 파라미터 컨트롤 수
  - 파라미터 이름
  - 슬라이더의 최소/최대/스텝 값
  - 컨트롤 유형 (슬라이더, 입력 등)
- 색상 섹션 (선택적):
  - 일부 아트는 색상 피커 필요
  - 일부 아트는 고정 색상 사용 가능
  - 일부 아트는 단색일 수 있음 (색상 컨트롤 불필요)
  - 아트의 필요에 따라 결정

**모든 아트워크는 고유한 파라미터와 알고리즘을 가져야 합니다!** 고정 부분은 일관된 UX를 제공하고 — 그 외 모든 것은 고유한 비전을 표현합니다.

### 필수 기능

**1. 파라미터 컨트롤**
- 숫자 파라미터용 슬라이더 (파티클 수, 노이즈 스케일, 속도 등)
- 팔레트 색상용 색상 피커
- 파라미터 변경 시 실시간 업데이트
- 기본값 복원을 위한 리셋 버튼

**2. 시드 내비게이션**
- 현재 시드 번호 표시
- 시드를 순환하는 "이전" 및 "다음" 버튼
- 랜덤 시드를 위한 "랜덤" 버튼
- 특정 시드로 이동하는 입력 필드
- 요청 시 100가지 변형 생성 (시드 1-100)

**3. 단일 아티팩트 구조**
```html
<!DOCTYPE html>
<html>
<head>
  <!-- CDN에서 p5.js — 항상 사용 가능 -->
  <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.7.0/p5.min.js"></script>
  <style>
    /* 모든 스타일링 인라인 — 깔끔하고 최소화 */
    /* 위에 캔버스, 아래에 컨트롤 */
  </style>
</head>
<body>
  <div id="canvas-container"></div>
  <div id="controls">
    <!-- 모든 파라미터 컨트롤 -->
  </div>
  <script>
    // 모든 p5.js 코드 여기에 인라인
    // 파라미터 객체, 클래스, 함수
    // setup()과 draw()
    // UI 핸들러
    // 모든 것이 자기 완결형
  </script>
</body>
</html>
```

**중요**: 단일 아티팩트입니다. 외부 파일 없음, 임포트 없음 (p5.js CDN 제외). 모든 것이 인라인.

**4. 구현 세부사항 — 사이드바 구축**

사이드바 구조:

**1. 시드 (고정)** — 항상 그대로 포함:
- 시드 표시
- 이전/다음/랜덤/이동 버튼

**2. 파라미터 (변수)** — 아트를 위한 컨트롤 생성:
```html
<div class="control-group">
    <label>파라미터 이름</label>
    <input type="range" id="param" min="..." max="..." step="..." value="..." oninput="updateParam('param', this.value)">
    <span class="value-display" id="param-value">...</span>
</div>
```
파라미터 수만큼 control-group div를 추가하세요.

**3. 색상 (선택적/변수)** — 아트에 조정 가능한 색상이 필요한 경우 포함:
- 사용자가 팔레트를 제어해야 하는 경우 색상 피커 추가
- 아트가 고정 색상을 사용하는 경우 이 섹션 생략
- 아트가 단색인 경우 생략

**4. 액션 (고정)** — 항상 그대로 포함:
- 재생성 버튼
- 리셋 버튼
- PNG 다운로드 버튼

**요구사항**:
- 시드 컨트롤이 작동해야 함 (이전/다음/랜덤/이동/표시)
- 모든 파라미터에 UI 컨트롤이 있어야 함
- 재생성, 리셋, 다운로드 버튼이 작동해야 함
- Anthropic 브랜딩 유지 (UI 스타일링, 아트 색상이 아님)

### 아티팩트 사용

HTML 아티팩트는 즉시 작동합니다:
1. **claude.ai에서**: 인터랙티브 아티팩트로 표시 — 즉시 실행
2. **파일로**: 저장하여 모든 브라우저에서 열기 — 서버 불필요
3. **공유**: HTML 파일 전송 — 완전히 자기 완결형

## 창의적 과정

**사용자 요청** → **알고리즘 철학** → **구현**

각 요청은 고유합니다. 과정은 다음을 포함합니다:

1. **사용자 의도 해석** — 어떤 미학을 추구하는가?
2. **알고리즘 철학 생성** (4-6 단락) 컴퓨테이셔널 접근 방식 설명
3. **코드로 구현** — 이 철학을 표현하는 알고리즘 구축
4. **적절한 파라미터 설계** — 무엇을 조정 가능하게 해야 하는가?
5. **매칭 UI 컨트롤 구축** — 해당 파라미터를 위한 슬라이더/입력

**상수**:
- Anthropic 브랜딩 (색상, 폰트, 레이아웃)
- 시드 내비게이션 (항상 존재)
- 자기 완결형 HTML 아티팩트

**나머지는 모두 변수**:
- 알고리즘 자체
- 파라미터
- UI 컨트롤
- 시각적 결과

최상의 결과를 위해 창의성을 믿고 철학이 구현을 안내하도록 하세요.
