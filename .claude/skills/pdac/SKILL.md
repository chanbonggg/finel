# Plan-Do-Analyze-Commit (PDAC) Workflow

자동화된 개발 워크플로우: 계획 → 구현 → 분석 → 커밋

## 개요

이 스킬은 4단계 개발 워크플로우를 자동화합니다:

1. **Plan**: planner 에이전트로 구현 계획 수립
2. **Do**: coder 에이전트로 코드 구현
3. **Analyze**: reviewer 에이전트로 코드 분석
4. **Commit**: /commit 스킬로 변경사항 커밋 및 푸시

## 사용방법

```
/pdac
```

사용자 입력을 받은 후 자동으로 다음 단계를 진행합니다:

1. 작업 설명 입력
2. planner 에이전트 호출 → 계획 수립
3. 사용자 승인 대기
4. coder 에이전트 호출 → 구현
5. reviewer 에이전트 호출 → 분석
6. /commit 스킬 호출 → 커밋 및 푸시

## 예시

```
/pdac

사용자: "Contact 폼에 파일 업로드 기능 추가"

[Step 1] Planner 에이전트 시작
→ 구현 계획 제시
→ "이 계획으로 진행하시겠습니까?" 질문

[사용자 승인 후]

[Step 2] Coder 에이전트 시작
→ 코드 구현

[Step 3] Reviewer 에이전트 시작
→ 코드 분석 및 피드백, 분석 내용을 자세히 설명

[Step 4] Commit 스킬 실행
→ 커밋 메시지 작성
→ git 커밋 및 푸시

완료!
```

## 워크플로우 흐름

```
사용자 입력
    ↓
[planner] Plan 단계
    ↓
사용자 승인?
    ↓ Yes
[coder] Do 단계
    ↓
[reviewer] Analyze 단계
    ↓
[/commit] Commit 단계
    ↓
완료
```

## 특징

- 🔄 **자동화**: 각 단계가 순차적으로 자동 진행
- ⏸️ **사용자 개입**: Plan 단계 후 승인 필요
- 🔍 **품질 보증**: Reviewer가 코드 품질 검사
- 📝 **자동 커밋**: 최종 커밋 및 푸시 자동화

## 주의사항

- Plan 단계에서 계획을 검토하고 승인해야 다음 단계 진행
- 각 단계는 독립적으로 진행되며, 실패 시 해당 단계 재시도 가능
- 최종 커밋 전에 모든 변경사항이 완료되어야 함
