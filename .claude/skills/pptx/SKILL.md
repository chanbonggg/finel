## 디자인 아이디어

**평범한 슬라이드는 만들지 마세요.** 흰 배경에 단순한 불릿 포인트는 누구에게도 인상을 주지 못합니다. 각 슬라이드에 아래 목록의 아이디어를 고려하세요.

### 시작 전에

- **굵고 주제에 맞는 색상 팔레트 선택**: 이 주제를 위해 특별히 디자인된 느낌이어야 합니다. 완전히 다른 프레젠테이션에 색상을 그대로 적용해도 '잘 맞는다'면, 충분히 특색 있는 선택이 아닙니다.
- **균형보다 지배**: 하나의 색이 지배적이어야 합니다 (시각적 비중 60-70%), 1-2개의 보조 톤과 하나의 강한 포인트 색상을 사용하세요. 모든 색상에 동등한 비중을 주면 안 됩니다.
- **명암 대비**: 제목 및 결론 슬라이드는 어두운 배경, 내용 슬라이드는 밝은 배경('샌드위치' 구조). 또는 프리미엄 느낌을 위해 전체를 어둡게 유지.
- **하나의 시각적 모티프에 집중**: 둥근 이미지 프레임, 색상 원 안의 아이콘, 두꺼운 단일 측면 테두리 등 뚜렷한 요소 하나를 선택하여 모든 슬라이드에 일관되게 적용하세요.

### 색상 팔레트

주제에 맞는 색상을 선택하세요 — 기본적인 파란색을 기본값으로 사용하지 마세요. 다음 팔레트를 참고하세요.

| 테마 | 주 색상 | 보조 색상 | 포인트 |
|-------|---------|-----------|--------|
| **미드나잇 이그제큐티브** | `1E2761` (남색) | `CADCFC` (아이스 블루) | `FFFFFF` (흰색) |
| **숲과 이끼** | `2C5F2D` (초록) | `97BC62` (이끼) | `F5F5F5` (크림) |
| **코랄 에너지** | `F96167` (코랄) | `F9E795` (골드) | `2F3C7E` (남색) |
| **따뜻한 테라코타** | `B85042` (테라코타) | `E7E8D1` (샌드) | `A7BEAE` (세이지) |
| **오션 그라데이션** | `065A82` (딥 블루) | `1C7293` (틸) | `21295C` (미드나잇) |
| **차콜 미니멀** | `36454F` (차콜) | `F2F2F2` (오프화이트) | `212121` (블랙) |

| **베리 앤 크림** | `6D2E46` (베리) | `A26769` (더스티 로즈) | `ECE2D0` (크림) |
| **세이지 캄** | `84B59F` (세이지) | `69A297` (유칼립투스) | `50808E` (슬레이트) |
| **체리 볼드** | `990011` (체리) | `FCF6F5` (오프화이트) | `2F3C7E` (남색) |

### 각 슬라이드별

**모든 슬라이드에는 시각적 요소가 필요합니다** — 이미지, 차트, 아이콘 또는 도형. 텍스트만 있는 슬라이드는 기억에 남지 않습니다.

**레이아웃 옵션:**
- 두 열 (텍스트 왼쪽, 일러스트 오른쪽)
- 아이콘 + 텍스트 행 (색상 원 안의 아이콘, 굵은 헤더, 아래 설명)
- 2x2 또는 2x3 그리드 (한쪽에 이미지, 다른 쪽에 콘텐츠 블록 그리드)
- 하프-블리드 이미지 (전체 왼쪽 또는 오른쪽)와 콘텐츠 오버레이

**데이터 표시:**
- 큰 통계 콜아웃 (60-72pt 큰 숫자와 아래에 작은 레이블)
- 비교 열 (전/후, 장단점, 나란히 옵션)
- 타임라인 또는 프로세스 흐름 (번호 단계, 화살표)

**시각적 세련됨:**
- 섹션 헤더 옆 작은 색상 원 안의 아이콘
- 주요 통계나 태그라인에 이탤릭 강조 텍스트

### 타이포그래피

**흥미로운 폰트 조합을 선택하세요** — Arial을 기본값으로 사용하지 마세요. 개성 있는 헤더 폰트를 선택하고 깔끔한 본문 폰트와 조합하세요.

| 헤더 폰트 | 본문 폰트 |
|-------------|-----------|
| Georgia | Calibri |
| Arial Black | Arial |
| Calibri | Calibri Light |
| Cambria | Calibri |
| Trebuchet MS | Calibri |
| Impact | Arial |
| Palatino | Garamond |
| Consolas | Calibri |

| 요소 | 크기 |
|---------|------|
| 슬라이드 제목 | 36-44pt 굵게 |
| 섹션 헤더 | 20-24pt 굵게 |
| 본문 텍스트 | 14-16pt |
| 캡션 | 10-12pt 흐리게 |

### 여백

- 최소 0.5인치 마진
- 콘텐츠 블록 사이 0.3-0.5인치
- 여유 공간 확보 — 모든 공간을 채우지 마세요

### 피할 것 (일반적인 실수)

- **같은 레이아웃 반복 금지** — 슬라이드 전체에 걸쳐 열, 카드, 콜아웃을 다양하게 변주하세요
- **본문 텍스트 가운데 정렬 금지** — 단락과 목록은 왼쪽 정렬; 제목만 가운데 정렬
- **크기 대비 소홀 금지** — 제목은 14-16pt 본문에서 두드러지려면 36pt 이상이어야 합니다
- **파란색 기본값 사용 금지** — 특정 주제를 반영하는 색상을 선택하세요
- **여백을 무작위로 사용 금지** — 0.3인치 또는 0.5인치 간격을 선택하고 일관되게 사용하세요
- **한 슬라이드만 스타일링하고 나머지는 평범하게 놔두지 마세요** — 완전히 적용하거나 처음부터 단순하게 유지하세요
- **텍스트만 있는 슬라이드 금지** — 이미지, 아이콘, 차트 또는 시각적 요소 추가; 단순한 제목 + 불릿 피하기
- **텍스트 박스 패딩 간과 금지** — 도형 또는 텍스트 가장자리와 선을 정렬할 때 텍스트 박스의 `margin: 0` 설정 또는 패딩을 고려하여 도형 오프셋 조정
- **저대비 요소 사용 금지** — 아이콘과 텍스트 모두 배경에 대해 강한 대비가 필요합니다; 밝은 배경에 연한 텍스트나 어두운 배경에 어두운 텍스트 사용 금지
- **제목 아래 강조선 절대 사용 금지** — 이는 AI 생성 슬라이드의 특징입니다; 대신 여백 또는 배경색을 사용하세요

## QA (필수)

**문제가 있다고 가정하세요. 여러분의 역할은 그것을 찾아내는 것입니다.**

첫 번째 렌더링은 거의 올바르지 않습니다. QA를 확인 단계가 아닌 버그 사냥으로 접근하세요. 첫 검사에서 문제를 하나도 찾지 못했다면, 충분히 꼼꼼하게 보지 않은 것입니다.

### 내용 QA

```bash
python -m markitdown output.pptx
```

누락된 내용, 오타, 잘못된 순서를 확인하세요.

**템플릿 사용 시 남은 플레이스홀더 텍스트 확인:**

```bash
python -m markitdown output.pptx | grep -iE "xxxx|lorem|ipsum|this.*(page|slide).*layout"
```

grep 결과가 나오면 완료를 선언하기 전에 수정하세요.

### 시각적 QA

**서브에이전트를 사용하세요** — 슬라이드가 2-3개라도. 코드를 계속 보고 있으면 실제로 있는 것이 아닌 기대하는 것을 보게 됩니다. 서브에이전트는 새로운 시각으로 봅니다.

슬라이드를 이미지로 변환한 후 ([이미지 변환](#이미지-변환) 참조), 다음 프롬프트를 사용하세요.

```
이 슬라이드를 시각적으로 검사하세요. 문제가 있다고 가정하고 찾아내세요.

다음을 확인하세요:
- 겹치는 요소 (도형을 통과하는 텍스트, 단어를 가로지르는 선, 쌓인 요소)
- 가장자리/박스 경계에서 잘리거나 넘치는 텍스트
- 한 줄 텍스트용으로 배치된 장식선이지만 제목이 두 줄로 줄 바꿈된 경우
- 위 내용과 충돌하는 출처 인용 또는 풋터
- 너무 가까운 요소 (< 0.3인치 간격) 또는 카드/섹션이 거의 붙어있는 경우
- 불균등한 간격 (한쪽은 큰 빈 공간, 다른 쪽은 빽빽한 경우)
- 슬라이드 가장자리에서 불충분한 마진 (< 0.5인치)
- 열 또는 유사한 요소가 일관되게 정렬되지 않은 경우
- 저대비 텍스트 (예: 크림색 배경에 연한 회색 텍스트)
- 저대비 아이콘 (예: 대비 원 없이 어두운 배경에 어두운 아이콘)
- 텍스트 박스가 너무 좁아 과도한 줄 바꿈 발생
- 남은 플레이스홀더 내용

각 슬라이드에 대해 사소한 것도 포함하여 문제 또는 우려 사항을 나열하세요.

다음 이미지를 읽고 분석하세요:
1. /path/to/slide-01.jpg (예상: [간략한 설명])
2. /path/to/slide-02.jpg (예상: [간략한 설명])

사소한 것을 포함하여 발견된 모든 문제를 보고하세요.
```

### 검증 루프

1. 슬라이드 생성 → 이미지 변환 → 검사
2. **발견된 문제 목록화** (아무것도 찾지 못했다면, 더 꼼꼼하게 다시 보세요)
3. 문제 수정
4. **영향 받은 슬라이드 재검증** — 하나의 수정이 종종 다른 문제를 만듭니다
5. 전체 검사에서 새로운 문제가 나오지 않을 때까지 반복

**수정 및 검증 사이클을 최소 한 번 완료할 때까지 완료 선언 금지.**

## 이미지 변환

시각적 검사를 위해 프레젠테이션을 개별 슬라이드 이미지로 변환합니다.

```bash
python scripts/office/soffice.py --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
```

`slide-01.jpg`, `slide-02.jpg` 등이 생성됩니다.

수정 후 특정 슬라이드를 재렌더링하려면:

```bash
pdftoppm -jpeg -r 150 -f N -l N output.pdf slide-fixed
```

## 의존성

- `pip install "markitdown[pptx]"` - 텍스트 추출
- `pip install Pillow` - 썸네일 그리드
- `npm install -g pptxgenjs` - 처음부터 생성
- LibreOffice (`soffice`) - PDF 변환 (`scripts/office/soffice.py`를 통해 샌드박스 환경에 자동 설정)
- Poppler (`pdftoppm`) - PDF를 이미지로 변환
